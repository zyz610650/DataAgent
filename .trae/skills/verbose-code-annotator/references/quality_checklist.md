# 注释质量自检清单 (Quality Checklist)

在输出最终携带注释的代码前，Agent 必须强制执行以下自我校验（内部思考过程，勿输出给用户）：

- [ ] **代码无损性校验**: 原始代码的任何业务逻辑、变量命名、语法结构是否均未被改变？
- [ ] **遗留注释保留校验**: 原有代码中的旧注释是否完好保留，未被覆盖或擅自删除？
- [ ] **文件头完整性**: 文件顶部是否添加了符合模板规范的头注释（职责、依赖）？
- [ ] **函数级文档完整性**: 核心业务函数是否详尽包含了参数、返回值、业务副作用及并发考量的说明？
- [ ] **废话滤除**: 是否杜绝了形如 `// i 加 1`、`// 返回 true` 等无意义废话注释？（必须解释“为什么”而不是“是什么”）
- [ ] **知识点与外链注入**: 对于识别到的中间件、数据库、并发逻辑，是否打上了 `【知识点】` 标签并附带了官方 `📚 参考文档` 链接？
- [ ] **白话化与受众意识**: 注释的表述是否足够通俗易懂，做到了“假设读者是刚毕业的实习生”级别？
- [ ] **格式对齐校验**: 插入的注释符号是否完全符合目标代码的语言规范（如 Go 使用 `//`，Python 使用 `#`）？排版缩进是否正确？

一定要记住:
1. 没完成所有注释和文档之前不允许终止
2.结束后检查所有后端代码是否添加了注解
3.结束后校验学习教程的合理和完整性
